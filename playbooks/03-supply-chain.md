# Playbook 3: Supply-Chain Compromise via AI-Generated Code

## Threat Model

LLMs introduce novel supply-chain risks:
- **Hallucinated packages:** Models suggest non-existent npm/PyPI packages, attackers register them with malware ("slopsquatting")
- **Backdoored suggestions:** Compromised model or RAG poisoning leads to suggestions that include subtle backdoors
- **Trusted-by-default code:** Developers paste AI-generated code without review
- **MCP server packages:** Attacker publishes malicious `@scope/mcp-server-x` that gains tool access on install

## Indicators

| Indicator | Where to look |
|-----------|---------------|
| Recently added packages with low download count | `package.json`, `requirements.txt`, `pyproject.toml` |
| Packages with names similar to popular ones (typosquatting) | Dependency trees |
| New MCP server entries pointing to npm packages | `claude_desktop_config.json` |
| Outbound connections from build agents to suspicious hosts | CI/CD logs |
| Large commits authored from AI tools | Git log |
| Code with subtle obfuscation in AI-suggested files | Static analysis |

## Triage Steps

### 1. Inventory AI-Authored Code
```bash
# Find git commits with AI tool signatures
git log --all --grep="Generated by Copilot\|Co-Authored-By: Claude\|cursor" --oneline

# Find files modified by AI tools (check IDE plugin logs)
python3 llm_triage.py --output ./evidence
cat ./evidence/timeline.json | jq '.timeline[] | select(.tool=="GitHub Copilot" or .tool=="Cursor")'
```

### 2. Diff Recent Dependency Changes
```bash
# Show recent additions to dependency files
git log -p --since="30 days ago" -- package.json package-lock.json requirements.txt pyproject.toml poetry.lock

# Check for hallucinated/squatted packages
npm view <package-name> downloads
pip show <package-name>
```

### 3. Audit MCP Server Sources
```bash
# Read the MCP config
cat ~/Library/Application\ Support/Claude/claude_desktop_config.json | jq '.mcpServers'

# For each entry, verify the npm package origin:
npm view @modelcontextprotocol/server-filesystem maintainers
npm view @modelcontextprotocol/server-filesystem repository
```

**Red flag:** MCP servers from unknown npm scopes, GitHub repos with no stars/history, or `command: node` pointing to local files in `/tmp` or download directories.

### 4. Static Analysis on AI-Generated Code
```bash
# Use semgrep with AI-suggestion-aware rules
semgrep --config=p/owasp-top-ten <repo>

# Look for common AI-suggestion vulnerabilities:
# - SQL injection from string concatenation
# - Command injection in subprocess.run/exec
# - Path traversal from user input
# - Hardcoded credentials in "example" code
# - Insecure deserialization (pickle, yaml.load)
```

## Containment

1. **Roll back the dependency change** — Revert `package.json`/`requirements.txt` to pre-incident state
2. **Rebuild from clean state** — Wipe `node_modules`, virtual environments, build caches
3. **Audit the production deployment** — Did the bad version reach prod?
4. **Rotate secrets** that the build environment had access to
5. **Disable MCP servers** that came from compromised sources
6. **Notify affected downstream consumers** if your code was published

## Detection Going Forward

- **SBOM enforcement:** Generate and verify Software Bill of Materials for every build
- **Package allowlisting:** Block install of packages not in approved registry
- **MCP server allowlist:** Maintain a corporate-approved list; alert on `claude_desktop_config.json` changes
- **AI code review gate:** Flag PRs touching dependency files for human security review
- **CI/CD egress controls:** Build agents should not have unrestricted internet access
- **Dependency pinning:** Lock files committed and verified in CI
- **Behavioral analysis:** Monitor for AI suggestions that introduce common vulnerability patterns (SQLi, command injection, hardcoded secrets) using pre-commit hooks
